Economic Policy Ideology CAT

This report documents a complete Computerized Adaptive Test (CAT) pipeline built around a 30-item forced-choice item bank measuring a single latent dimension of economic policy ideology (progressive ↔ conservative). Items were generated with AI assistance and span six subdomains: taxation, government spending, labor markets, healthcare, corporate regulation, and trade.

Why 2PL?

Each item presents two substantively grounded policy positions (option A = progressive, option B = conservative). Because neither option is "correct," there is no guessing parameter, making the three-parameter logistic model inappropriate. The two-parameter logistic (2PL) model captures variation in both the location (ideological threshold, parameter b) and slope (discriminating power, parameter a) of each item's characteristic curve.

Target population

English-speaking adults with an opinion on national economic policy. The latent trait theta is standardized so that theta = 0 corresponds to the ideological center; negative values indicate progressive orientation and positive values indicate conservative orientation.

Data Simulation

Five hundred respondents were drawn with theta ~ N(0, 1). For each person-item pair, the 2PL model determined P(B=1 | theta), and a binary response was Bernoulli-sampled. True item parameters were assigned deterministically from subdomain difficulty labels: Medium |b| ~ U(0, 0.5), Moderate |b| ~ U(0.5, 1.0), More extreme |b| ~ U(1.0, 1.8); sign alternated within each subdomain to spread information across the theta range. Discrimination a ~ U(0.8, 2.2). All draws used a fixed seed (42) for reproducibility.

Calibration

Item parameters were estimated via Marginal Maximum Likelihood EM using the girth library (twopl_mml). The algorithm marginalizes over theta using Gauss-Hermite quadrature and iterates until convergence. CAT Engine Design

Start rule

Begin with the item whose estimated difficulty b_hat is closest to 0 (ideological center) among items with above-median discrimination a_hat. This maximizes initial information for respondents near the population mean.

Item selection

At each step, compute Fisher information I(theta_hat) = a^2 * P(theta_hat) * (1 - P(theta_hat)) for every unadministered item and select the item with the highest value. Each item may be administered at most once per respondent.

Theta estimation

Theta is estimated by maximum likelihood (bounded Brent's method). When all responses so far are identical (e.g., all B or all A), MLE diverges; in that case, Expected A Posteriori estimation with a N(0, 1) prior is used as a fallback. The standard error is SE = 1 / sqrt(sum of item Fisher information at theta_hat).

Stop rule

Testing stops when SE < 0.30 (measurement precision achieved) or when 20 items have been administered (maximum length), whichever comes first.

Answer each question by clicking option A or option B. The test will stop when measurement precision is sufficient (SE < 0.30) or after 20 items. Your final ideology estimate will appear at the end.

Conclusion

Limitations

  • Small item bank: Thirty items is the lower bound for a credible CAT. The item pool is quickly exhausted for extreme respondents who may require many items to reach SE < 0.30.
  • Single dimension: Economic ideology is multidimensional in practice (e.g., fiscal conservatism vs. social conservatism). The unidimensional 2PL model is an approximation.
  • Simulation assumptions: True parameters were set manually and responses were drawn from the exact 2PL model. Real response data deviates from this ideal due to acquiescence bias, random responding, and context effects.
  • MLE instability: Early in the CAT (1-2 items administered) with uniform responses, MLE diverges. The EAP fallback mitigates this but introduces prior dependence.
  • Binary forced choice: Forced-choice format eliminates acquiescence bias but may frustrate respondents who hold nuanced positions and limits the response scale to a single bit of information per item.

Challenges encountered

  • Balancing item difficulty across subdomains while maintaining alternating b-parameter signs required careful constraint design.
  • Ensuring the interactive CAT's client-side Newton-Raphson converges reliably required replacing pure gradient descent with a bounded grid-search initialization.
  • Self-contained HTML with no server required inlining all Plotly figure data, demo step data, and calibrated parameters, making the file large but fully portable.
